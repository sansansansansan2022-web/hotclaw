# AI 阅读说明

开始任何代码生成、修改、重构、补丁、接口设计、表结构调整前，请先完整阅读项目根目录下的 `NOTICE.md`，并将其视为本项目的最高约束说明。

执行要求如下：

1. `NOTICE.md` 中的目录结构、技术栈、分层职责、输入输出协议、数据库约束、异常处理、日志要求、测试要求，默认全部生效。
2. 若当前任务与 `NOTICE.md` 冲突，以 `NOTICE.md` 为准，除非我在本次任务中明确说明要覆盖某一条规则。
3. 你只能做本次明确要求的范围，不要擅自扩展需求、改技术栈、改目录结构、改命名、加复杂抽象或提前实现未来功能。
4. 默认采用 MVP 最小可运行方案，优先保证：可运行、可维护、可追踪、可测试。
5. 输出代码前，先检查是否违反 `NOTICE.md`；如果有冲突，先指出冲突点，再给出最小修改方案。
6. 输出结果时优先给：
   - 新增/修改文件清单
   - 完整代码
   - 依赖变更
   - migration 建议
   - 最小测试用例
7. 若需求不完整，不要自由发挥，请基于 `NOTICE.md` 按最小闭环实现，并明确写出你的假设。

简化理解：
- 先读 `NOTICE.md`
- 严格按现有项目规范做
- 只做本次任务
- 不要擅自“优化”项目
