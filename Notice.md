## 1. 项目定位

这是一个**面向公众号内容生产的多智能体平台**，当前只实现 **MVP 第一版**。

系统目标：

- 用户最小输入：只输入“账号定位”
- 系统最小处理链：
  - 账号定位解析
  - 热点分析
  - 选题策划
  - 标题生成
  - 正文生成
  - 审核评估
- 输出：
  - 热点候选
  - 选题建议
  - 标题候选
  - 正文草稿
  - 审核结果
  - 可视化运行过程

当前阶段重点：

- 先做**可运行的最小闭环**
- 先做**草稿生成**
- 不追求全自动发布
- 不追求复杂自治 Agent
- 不追求超前平台能力

## 6. Agent / Skill 定义边界

### Agent
Agent 是工作流中的一个**节点角色**，负责完成单一业务任务，例如：

- 账号定位解析 Agent
- 热点分析 Agent
- 选题策划 Agent
- 标题生成 Agent
- 正文生成 Agent
- 审核 Agent

Agent 特点：

- 是工作流节点
- 有明确输入输出
- 尽量返回结构化 JSON
- 可以调用 Skill
- 不应承担多个职责

### Skill
Skill 是被 Agent 调用的**工具能力**，例如：

- 新闻抓取
- 文本摘要
- 风险检测
- 标题评分

Skill 特点：

- 不是工作流节点
- 不单独参与编排
- 只做工具型处理
- 输出尽量稳定可复用

### 禁止混淆
- 不要把 Agent 当 Skill 写
- 不要把 Skill 当工作流节点写
- 不要让 Agent 自主决定整个流程
- 第一版不做复杂自治 Agent

## 7. Workflow 与执行原则

当前工作流默认固定，不实现复杂拖拽和自治规划。

MVP 推荐链路：

1. 账号定位解析
2. 热点分析
3. 选题策划
4. 标题生成
5. 正文生成
6. 审核评估

要求：

- Agent 顺序由 Orchestrator 控制
- Agent 不得擅自跳过或新增步骤
- 单个节点失败时必须有明确错误输出
- 必须记录节点执行日志
- 必须支持任务级追踪

## 8. 输入输出协议要求

### 8.1 接口统一返回结构
所有 API 接口统一返回：

```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
````

错误时：

```json
{
  "code": 1001,
  "message": "validation error",
  "data": null,
  "details": {}
}
```

### 8.2 Agent 统一返回结构

所有 Agent 返回尽量统一为：

```json
{
  "status": "success",
  "agent_name": "topic_planner",
  "data": {},
  "error": null,
  "trace_id": "xxx"
}
```

失败时：

```json
{
  "status": "failed",
  "agent_name": "topic_planner",
  "data": null,
  "error": {
    "code": "TOPIC_GENERATION_FAILED",
    "message": "topic generation failed"
  },
  "trace_id": "xxx"
}
```

### 8.3 命名规则

* Python 变量、字段：`snake_case`
* TypeScript 类型、组件：`PascalCase`
* API path：小写、短横线或资源名统一风格
* 枚举值必须收敛，不允许任意字符串漂移

## 11. 错误处理要求

### 基本要求

* 所有外部调用必须设置超时
* 所有异常必须记录日志
* 所有节点失败必须返回明确错误
* 不允许裸 `except Exception: pass`
* 不允许吞掉异常后继续假装成功

### 异常分类

* 用户输入异常
* 配置异常
* 节点执行异常
* 第三方服务异常
* 系统异常

### 前后端要求

* 后端返回可解释错误码
* 前端正确处理 `loading / error / empty / success` 四态
* 不要把 traceback 原样暴露给用户

### 每个节点至少记录

* `task_id`
* `trace_id`
* `agent_name`
* `status`
* `started_at`
* `finished_at`
* `duration_ms`
* `input_snapshot`

* 错误日志与业务日志分开
