# 📬 马芸 ↔ qoder 双向通信协议

## 🎯 设计原则

- **2 个文件**：双方读写，形成循环
- **谁需要对方帮忙，谁写指令**
- **执行完写报告，等对方读取**
- **无需手动通知，定时轮询即可**

---

## 📁 文件结构

```
D:\project\hotclaw\backend\
├── instructions.md   ← 指令文件（谁需要对方做啥就写这里）
└── reports.md       ← 报告文件（执行结果写这里）
```

---

## 🔀 双向流程

```
┌─────────────────────────────────────────────────────────┐
│                      马芸 (你)                            │
│  收到你的需求 → 写指令到 instructions.md                   │
│  检查 reports.md 发现 qoder 的报告 → 反馈你              │
└─────────────────────────────────────────────────────────┘
                          ↓ read/write
┌─────────────────────────────────────────────────────────┐
│                    instructions.md                       │
│  [sender: san/mayun] [task description] [timestamp]    │
└─────────────────────────────────────────────────────────┘
                          ↓ read
┌─────────────────────────────────────────────────────────┐
│                      qoder                              │
│  读取指令 → 执行 → 写报告到 reports.md                    │
└─────────────────────────────────────────────────────────┘
                          ↓ read/write
┌─────────────────────────────────────────────────────────┐
│                     reports.md                           │
│  [sender: qoder] [result] [status] [timestamp]          │
└─────────────────────────────────────────────────────────┘
                          ↓ read
┌─────────────────────────────────────────────────────────┐
│                      马芸 (你)                            │
│  心跳时检查 reports.md → 发现新报告 → 反馈 san            │
└─────────────────────────────────────────────────────────┘
```

---

## ⚙️ 读写规则

### 马芸 的职责
| 时机 | 动作 |
|------|------|
| 收到 san 的需求 | 写指令到 `instructions.md`，标记 `[马芸] 发送给 qoder` |
| 心跳时 | 检查 `reports.md`，发现 qoder 的新报告就反馈 san |
| 发现 qoder 的求助指令 | 执行后写报告到 `reports.md`，标记 `[马芸] 执行报告` |

### qoder 的职责
| 时机 | 动作 |
|------|------|
| 定时检查 `instructions.md` | 发现马芸的指令就执行 |
| 执行完成 | 写报告到 `reports.md`，标记 `[qoder] 执行报告` |
| 有问题需要马芸帮忙 | 写指令到 `instructions.md`，标记 `[qoder] 发送给 马芸` |
| 发现马芸的求助指令 | 执行后写报告到 `reports.md` |

---

## 🔄 轮询机制

| 角色 | 检查频率 | 检查哪个文件 |
|------|----------|--------------|
| 马芸 | 心跳时 | `reports.md` |
| qoder | 每 30-60 秒 | `instructions.md` |
